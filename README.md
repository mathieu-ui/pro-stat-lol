# pro-stat-lol
Stat of lol player 
Python 3.12.3

La branche main de ce dépôt est protégée par un ensemble de règles afin d'assurer la stabilité, la sécurité et la qualité du code. Ces règles visent à contrôler les actions qui peuvent être effectuées sur cette branche, garantissant ainsi qu'aucune modification non autorisée ou accidentelle ne puisse altérer son contenu.
Règles de Protection Appliquées

    Interdiction de supprimer la branche main
        Règle activée : Les suppressions de la branche main sont désactivées.
        Détails : Seuls les utilisateurs disposant de permissions spéciales (permissions de contournement) peuvent supprimer cette branche. Cela garantit que la branche main ne peut pas être supprimée par erreur.

    Exiger une Pull Request avant de fusionner
        Règle activée : Avant de fusionner toute modification dans la branche main, il est obligatoire de passer par une pull request.
        Détails : Toutes les modifications doivent d'abord être apportées à une autre branche. Ensuite, elles doivent être soumises via une pull request, qui devra être revue et validée avant que la fusion dans la branche main puisse être effectuée. Cette règle permet de maintenir une bonne qualité de code et de valider chaque modification avant qu'elle ne soit intégrée à la branche principale.

    Interdiction des Pushs Forcés
        Règle activée : Les pushs forcés sont interdits sur la branche main.
        Détails : Les utilisateurs ayant un accès en écriture à la branche ne peuvent pas effectuer de pushs forcés, ce qui empêche la réécriture de l'historique des commits. Cette règle est importante pour garantir l'intégrité du code et éviter la perte d'historique ou l'introduction de modifications indésirables.

Objectifs de la Protection de la Branche main

La branche main étant le cœur du projet, il est essentiel de la protéger pour éviter les erreurs et garantir un flux de travail sécurisé. Ces règles visent à :

    Préserver l'intégrité du code : Seules les modifications validées et examinées peuvent être fusionnées, ce qui assure que le code dans main reste stable et fonctionnel.
    Assurer la sécurité du dépôt : Empêcher la suppression accidentelle de la branche et la réécriture de l'historique protège le projet contre les erreurs humaines et les manipulations malveillantes.
    Faciliter la collaboration : En exigeant une pull request avant toute fusion, tous les changements sont révisés par les membres de l'équipe, ce qui améliore la qualité et la cohérence du code.

Conclusion

Les règles appliquées à la branche main garantissent que cette branche, essentielle au bon fonctionnement du projet, reste protégée contre les actions non autorisées et les erreurs. Cela permet de maintenir un flux de travail collaboratif efficace et sécurisé, tout en assurant que les contributions apportées au code sont vérifiées et validées avant d'être intégrées à la branche principale.
